# 내 손안의 우주: 태양계 궤도 탐험기 화면 설계서

## 1. 설계 목표

- 초등학교 5~6학년 학생이 태양계 행성의 `순서`, `크기 차이`, `태양으로부터의 거리`, `공전 속도 차이`를 스스로 관찰하게 만든다.
- 첫인상은 `우주를 직접 탐험하는 느낌`이어야 하지만, 실제 수업에서는 `관찰 과제`가 분명히 보여야 한다.
- 한 화면에 모든 정보를 늘어놓지 않고, `먼저 관찰 -> 필요할 때 정보 확인 -> 다시 비교` 흐름이 자연스럽게 이어지도록 한다.

## 2. 대상과 사용 맥락

- 대상: 초등학교 5~6학년군 과학 수업의 학생
- 사용 맥락:
  - 교실 전자칠판/빔프로젝터 수업
  - 학생 개인 탐색 활동
  - 짧은 관찰 과제 중심의 자기주도 학습
- 연계 성취기준:
  - `[6과02-01]` 태양계를 구성하는 태양과 행성을 조사하고, 태양계 행성의 크기와 태양으로부터의 거리를 비교할 수 있다.

## 3. 최종 화면 방향

이번 설계에서 확정된 기본 방향은 아래와 같다.

- 메인 화면은 `몰입형 태양계 관찰 무대`로 구성한다.
- 행성 정보는 `오른쪽 슬라이드 패널`에서 열린다.
- 현재 해야 할 관찰 과제는 `상단 왼쪽 미션 칩`으로 짧게 노출한다.
- 시간 조절은 `하단 단순 조작 바`에서 처리한다.
- 화면 중앙은 가능한 한 비워 두고, 태양과 행성의 움직임이 가장 잘 보이게 한다.

## 4. 데스크톱 와이어프레임 구조

### 4-1. 상단 왼쪽: 미션 칩

- 현재 관찰해야 할 질문을 1문장으로 보여 준다.
- 예시:
  - `가장 빠르게 공전하는 행성을 찾아보세요.`
  - `태양에서 가장 먼 행성을 찾아보세요.`
- 문장은 짧고 행동 지시형으로 유지한다.
- 미션 칩은 항상 보이되, 시선을 과도하게 뺏지 않도록 작은 패널 형태로 유지한다.

### 4-2. 중앙: 태양계 관찰 무대

- 화면의 가장 큰 영역이다.
- 중앙에는 태양이 있고, 8개 행성이 각자의 궤도를 따라 공전한다.
- 행성은 클릭 가능한 대상이며, 클릭 시 오른쪽 정보 패널이 해당 행성 정보로 갱신된다.
- 궤도선은 학습 보조용으로 은은하게 보이고, 행성의 움직임을 읽기 쉽게 해야 한다.
- 실물 비율을 완전히 재현하지는 않으며, 아래 안내 원칙을 따른다.
  - `보기 쉽게 일부 크기와 거리 표현을 조정했습니다.`라는 학습용 축척 안내를 제공한다.

### 4-3. 오른쪽: 행성 정보 패널

- 선택한 행성의 세부 정보를 보여 주는 고정형 슬라이드 패널이다.
- 기본 상태:
  - 아무 행성도 선택하지 않았을 때는 `행성을 눌러 자세히 살펴보세요.` 같은 안내 문구를 보여 준다.
- 선택 상태:
  - 행성 이미지
  - 행성 이름
  - 한 줄 특징
  - 크기 정보
  - 태양으로부터의 거리 정보
  - 공전 주기 정보
  - 다음 탐색 유도 버튼
- 패널은 메인 무대를 가리지 않아야 하며, 비교 학습을 돕기 위해 메인 장면과 동시에 보이는 상태를 유지한다.

### 4-4. 하단: 단순 조작 바

- 하단 조작 바는 아래 4개 기능만 기본 제공한다.
  - `재생/일시정지`
  - `시간 빨리 감기 슬라이더`
  - `초기화`
  - `비교 보기`
- 이 조작 바의 역할은 학생에게 `시간을 바꾸어 보면 공전 차이가 보인다`는 학습 행동을 즉시 이해시키는 것이다.
- 기능 밀도를 낮춰 초등학생이 버튼 탐색에 시간을 쓰지 않게 한다.

## 5. 모바일 구조 원칙

- 메인 태양계 무대는 가장 크게 유지한다.
- 오른쪽 정보 패널은 모바일에서 `바텀 시트`로 내려온다.
- 미션 칩은 상단에 유지하되 더 짧은 문장으로 줄인다.
- 하단 조작 바는 한 줄 또는 두 줄로 단순 재배치한다.
- 모바일에서도 핵심 순서는 같다.
  - `관찰 -> 클릭 -> 정보 확인 -> 시간 조절`

## 6. 주요 사용자 흐름

### 6-1. 첫 진입

1. 학생은 태양과 8개 행성이 움직이는 화면을 본다.
2. 상단 미션 칩에서 오늘의 관찰 질문을 읽는다.
3. 하단 슬라이더나 재생 버튼을 보고 조작 가능하다는 점을 이해한다.

### 6-2. 행성 탐색

1. 학생이 특정 행성을 클릭한다.
2. 오른쪽 패널이 열리거나 갱신된다.
3. 학생은 해당 행성의 이미지와 특징을 읽는다.
4. 다시 메인 무대를 보며 다른 행성과 비교한다.

### 6-3. 시간 비교

1. 학생이 시간 슬라이더를 움직인다.
2. 공전 속도 차이가 즉시 시각화된다.
3. 미션 질문과 연결해 답을 스스로 찾는다.

### 6-4. 비교 보기

1. 학생이나 교사가 하단 조작 바의 `비교 보기`를 누른다.
2. 별도의 비교 레이어나 패널에서 `크기 비교` 또는 `거리 비교`를 본다.
3. 비교가 끝나면 다시 메인 무대로 돌아온다.

## 7. 정보 설계 원칙

- 한 번에 너무 많은 설명을 주지 않는다.
- 모든 설명은 초등학생 눈높이 문장으로 짧게 쓴다.
- 정보 패널의 우선순위는 아래 순서를 따른다.
  1. 이름
  2. 실제 사진 또는 질감 이미지
  3. 한 줄 특징
  4. 크기
  5. 거리
  6. 공전 주기
- 비교용 수치는 길게 나열하지 않고, `지구보다 큼/작음`, `태양에 가까움/멀음`, `공전이 빠름/느림`처럼 해석 가능한 문장과 함께 제시한다.

## 8. 학습 장치

- 미션 칩은 단순 장식이 아니라 학습 유도 장치다.
- 미션은 1회에 1개씩만 보여 준다.
- 추천 미션 유형:
  - 가장 빠르게 도는 행성 찾기
  - 태양에서 가장 먼 행성 찾기
  - 가장 큰 행성 찾기
  - 지구와 비슷한 크기 행성 찾기
- 학생이 화면을 조작하면서 자연스럽게 성취기준의 비교 활동을 수행하도록 유도한다.

## 9. 시각 연출 원칙

- 분위기는 `신비로운 우주`이되, 정보 가독성을 해치지 않아야 한다.
- 배경은 어두운 남색 계열, 별은 은은하게 처리한다.
- 태양은 가장 강한 시각 중심이어야 한다.
- 행성은 실제 사진 느낌을 살리되, 클릭 가능하다는 점이 보이게 해야 한다.
- 궤도선은 설명용 보조 요소이며 지나치게 강하게 보이면 안 된다.
- 전체 화면은 카드형 대시보드처럼 보이지 않아야 한다.

## 10. 오개념 방지 원칙

- 크기와 거리 표현은 한 화면에서 완전한 실제 축척을 재현하지 않는다.
- 따라서 반드시 `학습용 시각화`라는 안내를 포함한다.
- 비교 기능에서는 `크기 비교`와 `거리 비교`를 분리해서 보여 준다.
- 한 화면만 보고 `실제 우주도 이 정도 간격이다`라고 오해하지 않게 설계한다.

## 11. 접근성 및 수업 활용성

- 글자 크기는 초등학생과 교사가 멀리서 봐도 읽히는 수준이어야 한다.
- 색만으로 정보 차이를 전달하지 않는다.
- 키보드 포커스와 버튼 라벨을 제공한다.
- 애니메이션이 빠를 때도 UI 문구는 안정적으로 읽혀야 한다.
- 교실 발표 상황을 고려해 중요한 버튼은 넉넉한 크기로 설계한다.

## 12. 설계 완료 기준

- 학생이 첫 화면에서 `행성들이 태양 주위를 돈다`는 사실을 바로 이해할 수 있다.
- 상단 미션 칩만 보고도 무엇을 관찰해야 하는지 알 수 있다.
- 행성을 눌렀을 때 정보 확인이 자연스럽고, 메인 관찰 화면이 유지된다.
- 시간 조절이 쉽고, 공전 속도 차이가 즉시 체감된다.
- 비교 보기 기능이 크기와 거리 학습을 분리해 돕는다.
- 데스크톱과 모바일 모두에서 같은 학습 흐름이 유지된다.
